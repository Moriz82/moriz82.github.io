HTB Easy Linux Grafana Privilege Escalation

Description (parsed): Planning is an easy Linux machine emphasizing web recon, default creds, known Grafana exploitation, container context checks, and root escalation through local services.

Enumeration

Initial recon identifies nginx and SSH. Further web enumeration shows the primary site is mostly static/noisy, while ffuf discovers a Grafana subdomain as the real target surface.

nmap -sC -sV -Pn 10.10.x.x
ffuf -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt -u http://planning.htb -H \"Host: FUZZ.planning.htb\" -fs [baseline]

Foothold

Using provided Grafana creds, the report validates an outdated version and reproduces a public exploit chain. Initial shell context is root-like but constrained by container boundaries and missing flag access.

# references cited in report
https://github.com/nollium/CVE-2024-9264
https://www.revshells.com/

Container checks and environment review reveal plaintext credentials suitable for SSH pivot to the host, where user-level objectives are completed.

This writeup captures a common CTF pattern: root shell inside container first, then host pivot for real flag/impact.

Privilege Escalation

Post-host access, linPEAS highlights cron artifacts. A database file reveals additional plaintext credentials. Local port 8000 service is tunneled over SSH, and authenticated as root to gain privileged code execution.

linpeas.sh
cat crontab.db
ss -tulnp
ssh -L 8000:127.0.0.1:8000 user@10.10.x.x

The report closes with root-flag retrieval via permissions handling and notes alternate persistence/impact options for real-world engagements.

The original report explicitly reflects on better post-exploitation actions than only copying flags, which is useful for translating CTF workflow to client-like reporting.
